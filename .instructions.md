## Data Fetching

- **Always use helper functions in `/data`** for database access; never query `db` directly in components.
- All helpers must use Drizzle ORM and be marked with `'use server'` when needed.
- See [.github/instructions/data-fetching.instruction.md](.github/instructions/data-fetching.instruction.md).

## Key Rules (Non-Negotiable)

1. **Authentication**: Use Clerk only (`@clerk/nextjs/server`). Protect `/dashboard`. No custom auth.
2. **UI Components**: Use shadcn/ui primitives only (`components/ui/*`). No bespoke components.
3. **Data Access**: Always use `/data` helpers. No direct `db` queries in components.
4. **Server Components**: Prefer Server Components; add `'use client'` only when necessary.
5. **Fonts**: Use Nunito font for the entire app (already configured in `app/layout.tsx` and `app/globals.css`).
6. **Error Handling**: Wrap async operations in try-catch. Log errors, return safe defaults.
7. **Type Safety**: No `any` types without explanation. Type all props and returns explicitly.
